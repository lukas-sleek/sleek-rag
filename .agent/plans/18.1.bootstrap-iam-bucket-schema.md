## Tasks

### Task 1 — Add IAM roles to existing service account

**What:**
- The backend already authenticates via a service account JSON key (used today for Document AI Layout Parser). Identify the SA email from the existing key file or from `gcloud auth list`.
- Grant project-level Vertex AI access and bucket-level Storage access.

**Commands** (replace `PROJECT_ID` and `SA_EMAIL`):

```bash
# Vertex AI: corpus CRUD, file import, retrieval queries, Gemini inference
gcloud projects add-iam-policy-binding PROJECT_ID \
  --member="serviceAccount:SA_EMAIL" \
  --role="roles/aiplatform.user"

# Drop Document AI permission — LLM Parser inside RAG Engine doesn't need it
gcloud projects remove-iam-policy-binding PROJECT_ID \
  --member="serviceAccount:SA_EMAIL" \
  --role="roles/documentai.apiUser" \
  || echo "binding not present — skip"
```

(Storage role added in Task 2 after the bucket exists.)

**Validation:**
```bash
gcloud projects get-iam-policy PROJECT_ID \
  --flatten="bindings[].members" \
  --filter="bindings.members:SA_EMAIL" \
  --format="table(bindings.role)"
```
Expect to see `roles/aiplatform.user` listed. Expect `roles/documentai.apiUser` absent.

### Task 2 — Create GCS bucket

**What:**
Create one regional bucket in `europe-west3` to hold both ingestion sources and permanent original files (Q5 + Option B from Q2).

```bash
gcloud storage buckets create gs://sleek-rag-files-dev \
  --location=europe-west3 \
  --uniform-bucket-level-access \
  --soft-delete-duration=30d \
  --enable-versioning
```

Grant the SA object-admin on this bucket only (least privilege):

```bash
gcloud storage buckets add-iam-policy-binding gs://sleek-rag-files-dev \
  --member="serviceAccount:SA_EMAIL" \
  --role="roles/storage.objectAdmin"
```

**Layout convention** (used by 18.2 and 18.4):
```
gs://sleek-rag-files-dev/
  {user_id}/
    {project_id}/
      {file_id}/
        original.pdf      # canonical PDF (post-LibreOffice if needed)
        original.docx     # iff user uploaded DOCX (kept for reference, not parsed)
        original.xlsx     # iff user uploaded XLSX
```

**Validation:**
- `gcloud storage ls gs://sleek-rag-files-dev` → bucket exists, empty
- `gcloud storage cp /tmp/test.txt gs://sleek-rag-files-dev/test.txt` → succeeds with backend SA credentials
- `gcloud storage rm gs://sleek-rag-files-dev/test.txt` → succeeds

### Task 7 — Smoke test script

**What:**
New file `scripts/smoke_vertex_rag.py` — a standalone script that proves the IAM + bucket + SDK + project setup all work end-to-end before any app code touches them.

```python
"""Smoke test: create a corpus, upload a file, query, delete.
Run once after 18.1 to confirm GCP plumbing works."""

import os
import time
import vertexai
from vertexai.preview import rag
from google.cloud import storage

PROJECT_ID = os.environ["GCP_PROJECT_ID"]
LOCATION = os.environ["GCP_LOCATION"]
BUCKET = os.environ["GCS_FILES_BUCKET"]

vertexai.init(project=PROJECT_ID, location=LOCATION)

# 1. Upload a sample PDF
sample = "scripts/fixtures/smoke_sample.pdf"  # commit a small test PDF here
gcs_path = f"gs://{BUCKET}/_smoke/sample.pdf"
storage.Client().bucket(BUCKET).blob("_smoke/sample.pdf").upload_from_filename(sample)
print(f"Uploaded {sample} → {gcs_path}")

# 2. Create a corpus
corpus = rag.create_corpus(
    display_name="smoke-test",
    backend_config=rag.RagVectorDbConfig(
        rag_embedding_model_config=rag.RagEmbeddingModelConfig(
            vertex_prediction_endpoint=rag.VertexPredictionEndpoint(
                publisher_model="publishers/google/models/text-embedding-005"
            )
        )
    ),
)
print(f"Created corpus: {corpus.name}")

# 3. Import the file
op = rag.import_files(corpus.name, [gcs_path])
print(f"Import LRO: {op.name if hasattr(op, 'name') else op}")

# 4. Wait for ingestion (poll up to 5 minutes)
deadline = time.time() + 300
while time.time() < deadline:
    files = list(rag.list_files(corpus.name))
    if files and all(f.state.name == "ACTIVE" for f in files):
        print(f"All {len(files)} file(s) active")
        break
    print(f"Waiting on ingestion ({len(files)} files seen)...")
    time.sleep(15)
else:
    raise TimeoutError("Ingestion did not complete in 5 minutes")

# 5. Query
result = rag.retrieval_query(
    rag_resources=[rag.RagResource(rag_corpus=corpus.name)],
    text="What is this document about?",
    rag_retrieval_config=rag.RagRetrievalConfig(top_k=3),
)
print(f"Got {len(result.contexts.contexts)} contexts")
for ctx in result.contexts.contexts:
    print(f"  - score={ctx.score:.3f} text={ctx.text[:80]!r}")

# 6. Cleanup
rag.delete_corpus(corpus.name)
storage.Client().bucket(BUCKET).blob("_smoke/sample.pdf").delete()
print("Cleaned up. Smoke test passed.")
```

Add a small fixture PDF at `scripts/fixtures/smoke_sample.pdf` (any short PDF, ~1 page, ≤1 MB).

**Validation:**
- `python scripts/smoke_vertex_rag.py` → ends with `Smoke test passed.`
- No auth errors, no permission errors, no quota errors
- Cleanup is idempotent — running twice in a row works
