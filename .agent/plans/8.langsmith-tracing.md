# 8. LangSmith Tracing

✅ **Simple** — Wrap, don't instrument by hand.

## Context

CLAUDE.md mandates LangSmith from day one. After sub-plans 6 and 7, the backend has two OpenAI-touching surfaces (`responses.stream` in chats, `files.create` + `vector_stores.files.create_and_poll` in files). LangSmith's `wrap_openai` auto-traces every SDK call; `@traceable` decorates the route handlers so each user-initiated action becomes one parent run with the OpenAI calls as children.

## Goal

Send a chat message → see a parent trace `chat.send_message` in LangSmith with the streamed OpenAI call as a child span and the full input/output recorded. Same for file upload → trace `files.upload` with the file create + vector store poll as children.

## Prerequisites

- `LANGSMITH_API_KEY` in `backend/.env`.
- Sub-plans 6 + 7 landed.

## Tasks

### 8.1 Install + env

Append to `backend/requirements.txt`:
```
langsmith==0.1.147
```
Reinstall in venv.

`backend/.env` keys (already in `config.py` from sub-plan 4):
```
LANGSMITH_API_KEY=<from langsmith.com>
LANGSMITH_PROJECT=sleek-rag
```

### 8.2 Boot-time tracing config

In `backend/app/main.py`, before the `FastAPI()` line:
```python
import os
from app.config import settings

if settings.langsmith_api_key:
    os.environ["LANGSMITH_TRACING"] = "true"
    os.environ["LANGSMITH_API_KEY"] = settings.langsmith_api_key
    os.environ["LANGSMITH_PROJECT"] = settings.langsmith_project
```

(LangSmith reads these env vars at SDK call time. Setting via `os.environ` keeps `.env` as the single source of truth and lets us short-circuit when the key is empty.)

### 8.3 Wrap the OpenAI client

In `backend/app/openai_client.py`:
```python
from openai import OpenAI
from langsmith.wrappers import wrap_openai

from app.config import settings

_client = None

def openai_client():
    global _client
    if _client is None:
        raw = OpenAI(api_key=settings.openai_api_key)
        _client = wrap_openai(raw) if settings.langsmith_api_key else raw
    return _client
```

This auto-traces every `responses.*`, `files.*`, `vector_stores.*` call.

### 8.4 Decorate handlers

In `backend/app/routers/chats.py`:
```python
from langsmith import traceable

@traceable(run_type="chain", name="chat.send_message")
async def _run_send_message(chat_id, text, user_id):
    ...  # current send_message body, returns the streaming generator
```

Wrapping a streaming endpoint is fiddly — the parent run finishes when the function returns, but streaming continues afterward. Two options:
- **Option A (simpler):** Move the per-event `responses.stream` consumption into the traceable function and `yield` deltas. The auto-traced child span captures input/output. The parent run records duration + final concatenated text.
- **Option B:** Skip `@traceable` on the streaming route and rely solely on `wrap_openai`'s auto-tracing of `responses.stream`. Each chat message becomes one run (the OpenAI call) without a parent group. Simpler but loses the "one trace per user action" framing.

**Recommend A.** Concatenate deltas server-side, return the final text from the traceable, and emit deltas as side-effect SSE writes via a queue. If complexity creeps, fall back to B for M1 and revisit in M2.

In `backend/app/routers/files.py`:
```python
@router.post("/...")
@traceable(run_type="chain", name="files.upload")
async def upload_file(...):
    ...  # current upload logic
```
Non-streaming, so straight decoration works.

### 8.5 Add user metadata

Inside each traceable, attach user id for filtering in LangSmith:
```python
from langsmith import get_current_run_tree

run = get_current_run_tree()
if run:
    run.add_metadata({"user_id": user_id})
```

## Validation

1. Send a chat message via the UI.
2. Open https://smith.langchain.com/ → project `sleek-rag` → see a new trace `chat.send_message`.
3. Expand → see the auto-traced `openai.responses.stream` child span with input messages + output text.
4. Trace metadata includes `user_id`.
5. Upload a file → see a `files.upload` trace with `openai.files.create` and `openai.vector_stores.files.create_and_poll` as children.
6. Unset `LANGSMITH_API_KEY` and restart → app still works, no traces emitted, no errors.
