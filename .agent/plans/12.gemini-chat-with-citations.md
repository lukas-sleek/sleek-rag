## Architectural decisions

### Retrieval router (the rules)

The chat endpoint inspects the user's text and picks one or more retrieval modes:

| Pattern (regex) | Retrieval mode | Notes |
|---|---|---|
| `\b(page|seite|p\.)\s*(\d+)\b` | **Page filter:** `WHERE page_start <= N AND page_end >= N` | Returns all chunks on that page |
| `\b(Figure|Abbildung|Fig\.|Abb\.)\s*[\d.]+` | **Figure label filter:** `WHERE figure_label = $extracted` | Exact match; if no hit, fall back to vector |
| `\b(Abschnitt|section|chapter|Kapitel)\s*[\d.]+` | **Heading filter:** `WHERE heading_path && ARRAY[$extracted_prefix]` | Trigram match on heading path |
| `(?i)\b(drawing|diagram|zeichnung|skizze|figure|abbildung)\b` AND no specific reference | **Block-type filter + vector:** `WHERE block_type = 'figure'` plus vector search | Returns figure chunks ranked by semantic similarity |
| (default) | **Pure vector search** | Top-K = 8 from `document_chunks.embedding` cosine similarity |

Always restrict by `project_id` (RLS does the rest).

### Multimodal input rule

**Send image bytes to Gemini** when any retrieved chunk satisfies:
- `block_type = 'figure'` AND has a row in `chunk_images`, AND
- The user query was either (a) a figure-label match, (b) a page-filter on a page that has figures, or (c) explicitly mentions visual terms.

Cap at **3 images per turn** (Gemini cost + context window protection). Each image goes in as an OpenAI-format `content` part:
```python
{"role": "user", "content": [
    {"type": "text", "text": user_query_with_chunk_context},
    {"type": "image_url", "image_url": {"url": f"data:image/png;base64,{b64}"}},
]}
```

### Citation shape

Each assistant message persists `citations` JSONB:
```json
[
  {
    "chunk_id": "uuid",
    "file_id": "uuid",
    "filename": "pumpe-handbuch.pdf",
    "page_start": 17,
    "page_end": 17,
    "snippet": "Die hydraulische Druckkurve zeigt...",
    "figure_label": "Figure 3.6",
    "image_path": "{user_id}/{file_id}/{chunk_id}.png",
    "score": 0.83
  }
]
```

The frontend consumes this JSON to render chips + open the source PDF at the right page (plan 13).

### SSE frame format (extended)

Today: `data: {"delta": "..."}` only.

After this plan, the stream emits:
1. **One** `meta` frame at the start, containing the citations JSON (so the UI can render chips before the answer streams in):
   ```
   data: {"type":"meta","citations":[{...}]}
   ```
2. **Many** `delta` frames as before:
   ```
   data: {"type":"delta","content":"..."}
   ```
3. **One** `done` frame to signal completion:
   ```
   data: {"type":"done","message_id":"<uuid>"}
   ```

Existing frontend (app-shell.tsx:874-919) consumes `{delta}` shape. Add type-discrimination there.

```python
"""Hybrid retrieval over document_chunks. Returns ranked chunks + image refs."""
import re
from dataclasses import dataclass, field
from langsmith import traceable

from app.config import settings
from app.db import supabase
from app.gemini_client import gemini_client


@dataclass
class RetrievedChunk:
    chunk_id: str
    file_id: str
    filename: str
    project_id: str
    content: str
    page_start: int
    page_end: int
    figure_label: str | None
    block_type: str
    score: float
    image_path: str | None = None  # populated if chunk has a chunk_images row

    def to_citation(self) -> dict:
        snippet = self.content[:200] + ("…" if len(self.content) > 200 else "")
        return {
            "chunk_id": self.chunk_id, "file_id": self.file_id,
            "filename": self.filename, "page_start": self.page_start,
            "page_end": self.page_end, "snippet": snippet,
            "figure_label": self.figure_label, "image_path": self.image_path,
            "score": self.score,
        }


# --- pattern detection ---

_PAGE_RE = re.compile(r"\b(?:page|seite|p\.?)\s*(\d+)", re.I)
_FIGURE_RE = re.compile(r"\b(Figure|Abbildung|Fig\.?|Abb\.?)\s*([\d.]+)", re.I)
_SECTION_RE = re.compile(r"\b(?:Abschnitt|section|chapter|Kapitel)\s*([\d.]+)", re.I)
_VISUAL_RE = re.compile(r"(?i)\b(drawing|diagram|zeichnung|skizze|figure|abbildung|grafik|schaubild|technische\s*zeichnung)\b")


@traceable(run_type="retriever", name="retrieval.run")
def retrieve(
    *, query: str, project_id: str, user_id: str, top_k: int = 8,
) -> list[RetrievedChunk]:
    """Single entry point. Picks a strategy and returns ranked chunks."""
    page_match = _PAGE_RE.search(query)
    figure_match = _FIGURE_RE.search(query)
    section_match = _SECTION_RE.search(query)
    visual = bool(_VISUAL_RE.search(query))

    if figure_match:
        label = f"{figure_match.group(1).rstrip('.')} {figure_match.group(2)}"
        rows = _by_figure_label(label, project_id)
        if rows: return _attach_images(rows)

    if page_match:
        page = int(page_match.group(1))
        rows = _by_page(page, project_id)
        if rows: return _attach_images(rows)

    if section_match:
        rows = _by_heading(section_match.group(1), project_id)
        if rows: return _attach_images(rows[:top_k])

    # Vector path (with optional block_type filter for visual queries)
    block_filter = "figure" if visual else None
    rows = _by_vector(query, project_id, top_k, block_filter)
    return _attach_images(rows)


def _by_figure_label(label: str, project_id: str) -> list[RetrievedChunk]:
    res = (supabase().table("document_chunks")
           .select("id,file_id,project_id,content,page_start,page_end,figure_label,block_type,project_files(filename)")
           .eq("project_id", project_id).eq("figure_label", label).execute())
    return [_row_to_chunk(r, score=1.0) for r in (res.data or [])]


def _by_page(page: int, project_id: str) -> list[RetrievedChunk]:
    res = (supabase().table("document_chunks")
           .select("id,file_id,project_id,content,page_start,page_end,figure_label,block_type,project_files(filename)")
           .eq("project_id", project_id).lte("page_start", page).gte("page_end", page).execute())
    return [_row_to_chunk(r, score=1.0) for r in (res.data or [])]


def _by_heading(prefix: str, project_id: str) -> list[RetrievedChunk]:
    # Use raw SQL via rpc for array containment
    res = supabase().rpc("chunks_by_heading_prefix", {
        "p_project_id": project_id, "p_prefix": prefix
    }).execute()
    return [_row_to_chunk(r, score=0.9) for r in (res.data or [])]


def _by_vector(query: str, project_id: str, top_k: int, block_type: str | None) -> list[RetrievedChunk]:
    emb = gemini_client().embeddings.create(
        model=settings.gemini_embedding_model, input=query,
        dimensions=settings.gemini_embedding_dim,
    ).data[0].embedding
    res = supabase().rpc("match_chunks", {
        "p_project_id": project_id, "p_embedding": emb, "p_top_k": top_k,
        "p_block_type": block_type,
    }).execute()
    return [_row_to_chunk(r, score=r["similarity"]) for r in (res.data or [])]


def _attach_images(chunks: list[RetrievedChunk]) -> list[RetrievedChunk]:
    if not chunks: return []
    chunk_ids = [c.chunk_id for c in chunks]
    images = (supabase().table("chunk_images")
              .select("chunk_id,storage_path").in_("chunk_id", chunk_ids).execute()).data or []
    by_chunk = {i["chunk_id"]: i["storage_path"] for i in images}
    for c in chunks:
        c.image_path = by_chunk.get(c.chunk_id)
    return chunks


def _row_to_chunk(row: dict, score: float) -> RetrievedChunk:
    filename = (row.get("project_files") or {}).get("filename", "?")
    return RetrievedChunk(
        chunk_id=row["id"], file_id=row["file_id"], filename=filename,
        project_id=row["project_id"], content=row["content"],
        page_start=row["page_start"], page_end=row["page_end"],
        figure_label=row.get("figure_label"), block_type=row["block_type"], score=score,
    )
```

**Add SQL helpers** in `supabase/migrations/0005_retrieval_fns.sql`:

```sql
-- Cosine-similarity ANN with optional block_type filter.
create or replace function public.match_chunks(
  p_project_id uuid, p_embedding vector(768), p_top_k int default 8,
  p_block_type text default null
) returns table (
  id uuid, file_id uuid, project_id uuid, content text,
  page_start int, page_end int, figure_label text, block_type text,
  filename text, similarity float
) language sql stable as $$
  select c.id, c.file_id, c.project_id, c.content,
         c.page_start, c.page_end, c.figure_label, c.block_type,
         pf.filename,
         1 - (c.embedding <=> p_embedding) as similarity
  from public.document_chunks c
  join public.project_files pf on pf.id = c.file_id
  where c.project_id = p_project_id
    and (p_block_type is null or c.block_type = p_block_type)
  order by c.embedding <=> p_embedding
  limit p_top_k;
$$;

create or replace function public.chunks_by_heading_prefix(
  p_project_id uuid, p_prefix text
) returns table (
  id uuid, file_id uuid, project_id uuid, content text,
  page_start int, page_end int, figure_label text, block_type text, filename text
) language sql stable as $$
  select c.id, c.file_id, c.project_id, c.content,
         c.page_start, c.page_end, c.figure_label, c.block_type, pf.filename
  from public.document_chunks c
  join public.project_files pf on pf.id = c.file_id
  where c.project_id = p_project_id
    and exists (
      select 1 from unnest(c.heading_path) h where h ilike p_prefix || '%'
    );
$$;
```

### 12.2 — Build the prompt assembler

**File:** `/home/lukasthomas/sleek-rag/backend/app/prompt.py` (new)

```python
import base64
from app.db import supabase
from app.retrieval import RetrievedChunk

SYSTEM_PROMPT = """You are a technical RAG assistant. Answer based ONLY on the provided context chunks.
Cite sources by the chunk number in square brackets, e.g. [1] or [2].
If the answer is not in the context, say so.
When images are attached, refer to them as 'the figure' or by their label and explain what they show."""

MAX_IMAGES_PER_TURN = 3
MAX_IMAGE_BYTES = 4 * 1024 * 1024  # 4 MB Gemini hard cap per image


def build_messages(
    *, query: str, history: list[dict], chunks: list[RetrievedChunk],
) -> list[dict]:
    """Assemble OpenAI-format messages with optional image parts."""
    context_block = _format_context(chunks)
    image_parts = _maybe_image_parts(chunks)

    user_content: list[dict] | str
    if image_parts:
        user_content = [
            {"type": "text", "text": f"{context_block}\n\n---\n\nQuestion: {query}"},
            *image_parts,
        ]
    else:
        user_content = f"{context_block}\n\n---\n\nQuestion: {query}"

    return [
        {"role": "system", "content": SYSTEM_PROMPT},
        *history,                    # preserved as-is from chat_messages
        {"role": "user", "content": user_content},
    ]


def _format_context(chunks: list[RetrievedChunk]) -> str:
    if not chunks:
        return "(No context retrieved.)"
    lines = ["Context:"]
    for i, c in enumerate(chunks, 1):
        head = f"[{i}] {c.filename} p.{c.page_start}"
        if c.figure_label: head += f" — {c.figure_label}"
        lines.append(f"{head}\n{c.content}")
    return "\n\n".join(lines)


def _maybe_image_parts(chunks: list[RetrievedChunk]) -> list[dict]:
    out: list[dict] = []
    for c in chunks:
        if not c.image_path or len(out) >= MAX_IMAGES_PER_TURN: continue
        try:
            data = supabase().storage.from_("chunk-images").download(c.image_path)
            if len(data) > MAX_IMAGE_BYTES: continue
            b64 = base64.b64encode(data).decode("ascii")
            out.append({"type": "image_url", "image_url": {"url": f"data:image/png;base64,{b64}"}})
        except Exception:
            continue
    return out
```

import json, asyncio
from app.gemini_client import gemini_client
from app.retrieval import retrieve
from app.prompt import build_messages

@traceable(run_type="chain", name="chats.send_message")
async def _send_message_stream(
    *, chat: dict, text: str, chat_id: str, user_id: str, template: list[str] | None,
):
    project_id = chat["project_id"]

    # 1. Persist user message.
    user_msg = await asyncio.to_thread(lambda: supabase().table("chat_messages").insert({
        "chat_id": chat_id, "user_id": user_id, "role": "user", "content": text,
    }).execute().data[0])

    # 2. Load history (last N messages).
    history_rows = await asyncio.to_thread(lambda: supabase().table("chat_messages")
        .select("role,content").eq("chat_id", chat_id).neq("id", user_msg["id"])
        .order("created_at").limit(20).execute().data) or []
    history = [{"role": r["role"], "content": r["content"]} for r in history_rows]

    # 3. Retrieve.
    chunks = await asyncio.to_thread(retrieve,
        query=text, project_id=project_id, user_id=user_id, top_k=8)
    citations = [c.to_citation() for c in chunks]

    # 4. Emit meta frame with citations.
    yield f"data: {json.dumps({'type': 'meta', 'citations': citations})}\n\n"

    # 5. Build messages, call Gemini.
    messages = build_messages(query=text, history=history, chunks=chunks)
    tools: list[dict] = [PROJEKTANALYSE_TOOL, PROJEKTANALYSE_V2_TOOL]

    stream = await asyncio.to_thread(
        gemini_client().chat.completions.create,
        model=settings.gemini_chat_model,
        messages=messages,
        tools=tools,
        stream=True,
    )

    parts: list[str] = []
    triggered: str | None = None
    tool_call_buffer: dict | None = None

    try:
        for event in stream:
            choice = event.choices[0]
            delta = choice.delta
            if delta.content:
                parts.append(delta.content)
                yield f"data: {json.dumps({'type': 'delta', 'content': delta.content})}\n\n"
            if delta.tool_calls:
                tc = delta.tool_calls[0]
                if tc.function and tc.function.name:
                    name = tc.function.name
                    if name == "run_projektanalyse": triggered = "v1"; break
                    if name == "run_projektanalyse_v2": triggered = "v2"; break
    finally:
        await asyncio.to_thread(stream.close)

    # 6. Persist assistant message + citations.
    assistant_text = "".join(parts).strip()
    if assistant_text:
        msg = await asyncio.to_thread(lambda: supabase().table("chat_messages").insert({
            "chat_id": chat_id, "user_id": user_id, "role": "assistant",
            "content": assistant_text, "citations": citations,
        }).execute().data[0])
        yield f"data: {json.dumps({'type': 'done', 'message_id': msg['id']})}\n\n"
    else:
        yield "data: {\"type\":\"done\"}\n\n"

    # 7. Projektanalyse hand-off (unchanged behavior, adapted to new client).
    if triggered == "v1":
        async for sse in stream_projektanalyse(
            template=template, chat_id=chat_id, user_id=user_id,
        ): yield sse
    elif triggered == "v2":
        async for sse in stream_projektanalyse_v2(
            template=template, chat_id=chat_id, user_id=user_id,
        ): yield sse

## Risks / pitfalls

- **OpenAI-compat tool calling on Gemini:** verify with a smoke test before committing — Gemini's OpenAI-compat endpoint supports `tools=[...]`, but tool-call streaming chunk shape can differ subtly. If the projektanalyse trigger detection breaks, we may need to use Gemini's native `google-genai` SDK for that one call. Acceptable trade-off — the retrieval path stays OpenAI-shaped.
- **Embedding dimension mismatch:** if any chunks were embedded at a different dim during plan 11, `match_chunks` will error. Sanity-check `select distinct vector_dims(embedding) from document_chunks` returns a single row = 768.
- **Projektanalyse v2** sends file_ids to OpenAI; the new path needs to send chunks/text instead. Adapt `stream_projektanalyse_v2` to query `document_chunks` filtered by `file_id IN (...)`.

## Rollback

Revert this commit only — plans 10 and 11 stay in place. The OpenAI ingestion code in `files.py` is no longer reachable but harmless. Old chats remain readable from `chat_messages` (newly written ones); legacy `openai_thread_id` chats just show empty.
