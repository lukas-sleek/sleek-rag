## Goal

Replace the 18.3 Pattern A chat loop (single Vertex RAG-grounded Gemini call + byte-offset citation splicer in `backend/app/citations.py`) with a Google ADK multi-agent system:

```
chat_orchestrator (gemini-2.5-pro)
  ├─ tool: rag_specialist (AgentTool)             # one self-contained question
  │    └─ tool: document_retriever (AgentTool)
  │         └─ tool: search_project_documents  # plain FunctionTool wrapping rag.async_retrieve_contexts
  ├─ tool: web_researcher (AgentTool)
  │    ├─ tool: web_google_search (AgentTool)
  │    └─ tool: web_url_fetcher (AgentTool)
  └─ tool: run_projektanalyse_v2                   # FunctionTool, hands off to existing streamer
```

Three architectural decisions, each forced by research findings:

1. **Per-corpus `AdkApp` factory + LRU cache** — quality over runtime simplicity. Each project gets its own `AdkApp` with the corpus pre-bound at construction (LRU-cached, max 256 apps in process). Per-project agent specialization stays on the table for future work. (T0 probe 6 settled an earlier misconception: `app_name` is hard-coded to `"default-app-name"` by `AdkApp` — it is **not** a constructor parameter, and each `AdkApp` instance owns its own `InMemorySessionService`, so session_ids do **not** cross instances under in-memory storage. This is fine because strategy (c) below replays Supabase rows into a fresh per-turn session.) ([adk-factory-and-history.md §4.1, §2.4](../research/adk-factory-and-history.md), assumptions doc T0 probe 6/7)

2. **Custom `FunctionTool` wrapping `rag.async_retrieve_contexts` — DO NOT subclass `VertexAiRagRetrieval`.** The plan originally proposed subclassing the managed tool to override its metadata-discarding `run_async`. T0 probe 2 verified live that `VertexAiRagRetrieval.process_llm_request` registers the tool as a *server-side* `types.Tool(retrieval=vertex_rag_store)` for any Gemini 2.x model — Gemini calls Vertex RAG inside its own server, returns `grounding_metadata` (the same shape 18.3 already uses), and `run_async` is never invoked. The `ADK_DISABLE_GEMINI_MODEL_ID_CHECK=1` env var the original plan eyed for bypassing this **forces** the server-side path on every model; it does not bypass it. With no public hook to flip the path, the only way to get structured chunks back into `tool_context.state["citations"]` is a **plain `FunctionTool` callable that owns the retrieval directly**. We lose `process_llm_request` (which was producing the same byte-offset citation shape we are trying to escape — losing it is the *point*). The function declaration `search_project_documents(query)` is registered with Gemini, the LLM places `[N]` markers inline, and citations live in session state. ([assumptions doc T0 probe 2](../research/adk-migration-assumptions.md), [§1.3, §3.1, §3.3](../research/adk-factory-and-history.md))

3. **Replay Supabase `chat_messages` into a fresh `InMemorySessionService` per turn — defer Vertex Agent Engine Sessions to a follow-up phase.** Three options were evaluated: (a) `VertexAiSessionService` against an empty Agent Engine, (b) `DatabaseSessionService` against Supabase, (c) keep our `chat_messages` table as source of truth and replay each turn into in-memory. (b) is rejected because ADK owns its own schema (StorageSession/StorageEvent/...) which migrated at v1.22 and is incompatible with `chat_messages`. (a) is the long-term destination but requires a new Agent Engine resource per env — out of scope for 19.0. (c) ships zero new infra, keeps the existing chat-list UI working, and the migration to (a) later is mechanical (swap the `session_service_builder`). The cost: tool-call/response pairing across turns is opaque to ADK because we serialize prior turns as plain text — same penalty we already pay today. ([§2.5 Recommendation](../research/adk-factory-and-history.md))

Plus the design improvements that don't depend on the research:

4. **Query rephrasing moves to the orchestrator** (Pro-level pronoun resolution + multi-question split + compound-question synthesis without self-computation). The seven-rule routing block stays as designed.
5. **`agent_tool.AgentTool` instead of `sub_agents=`** at every edge so each invocation is an explicit call with a self-contained question argument, enabling multi-question fan-out.

After 19.0, the test suite shifts: integration tests now mock the `AdkApp` factory rather than `client.aio.chats.create`. A new eval set (`backend/tests/eval_followup_dispatch.py`) regression-tests orchestrator routing on pronoun resolution, compound, multi-question, and ambiguous-reference cases.

## Why now (and what 18.3 got wrong)

Pattern A is correct for self-contained questions and breaks down on:

- **Follow-ups** ("und das?", "wie viel davon entfällt auf X?") — no rephrasing layer; model either uses chat history correctly or hallucinates from it.
- **Multi-question turns** ("Was ist X und welche Y?") — one Vertex RAG call retrieves once for the merged question; relevance suffers for one or both sub-questions.
- **Citation placement** — `grounding_supports` byte-offset splicing puts `[N]` at sentence ends regardless of where the cited fact lives. ~25% of short answers had zero `grounding_supports` in production (LangSmith research §3).
- **Page-marker reliability** — LLM Parser's `[Seite N]` injection holds 89% of the time (LangSmith §2). The current regex contract handles `None` pages, but the failure mode lives in an answer-side regex pipeline that's hard to evolve.

ADK's agent tree fixes all four with a Pro-level routing layer, structured tool outputs as the citation source of truth, and explicit per-question dispatch.

### Task 2 — `make_search_project_documents_tool(corpus_name)` factory (plain `FunctionTool`)

**What:**

A factory that returns a `FunctionTool` whose backing async callable owns the Vertex RAG retrieval directly. Replaces the originally planned `VertexAiRagRetrieval` subclass (T0 probe 2 proved that path is unreachable for Gemini 2.x). Single regex pair for `[Seite N]` / `[Abb. N: …]` lives here at the retrieval boundary; never touches answer prose.

```python
# backend/app/adk/retrieval_tool.py — new
"""Custom FunctionTool that retrieves chunks from a per-project Vertex RAG corpus.

Pivot away from the originally planned `VertexAiRagRetrieval` subclass:
T0 probe 2 verified that the managed tool's `process_llm_request` always
registers a server-side `Tool(retrieval=...)` for Gemini 2.x — `run_async`
is never called, so an override is dead code. Instead we declare a plain
`FunctionTool` whose async callable:
  1. Calls `rag.async_retrieve_contexts` (non-blocking).
  2. Builds structured per-chunk records with regex-extracted page +
     figure metadata.
  3. Writes the records into `tool_context.state["citations"]` for the
     post-run aggregator in chats.py to dedupe + renumber.
  4. Returns a structured dict to the LLM so it can place [N] markers
     and reason about which file/page the chunk came from.

The corpus is closed over by the factory — one `FunctionTool` per cached
`AdkApp`. Closure-style binding is the simplest way to bind per-project
state to a `FunctionTool` (the alternative would be reading a corpus name
from `tool_context.state`, which adds an extra lookup and a failure mode).
"""
from __future__ import annotations

import re
from typing import Any

from google.adk.tools import FunctionTool, ToolContext
from langsmith import traceable
from vertexai.preview import rag


# Boundary regex — operates on parser output (`[Seite N]` / `[Abb. N: …]`
# markers injected by the LLM Parser during ingestion). Never applied to
# the LLM's answer prose.
_PAGE_RE = re.compile(r"\[Seite\s+(\d+)\]")
_FIGURE_RE = re.compile(r"\[Abb\.?\s*(\d+(?:\.\d+)*)\s*:\s*([^\]]+)\]")

_DEFAULT_TOP_K = 10


def make_search_project_documents_tool(
    corpus_name: str,
    *,
    top_k: int = _DEFAULT_TOP_K,
) -> FunctionTool:
    """Build a per-corpus `search_project_documents` FunctionTool.

    The returned tool exposes a single argument `query: str` to the LLM and
    yields chunks plus side-channel citations (in `tool_context.state`).
    """

    @traceable(run_type="retriever", name="search_project_documents")
    async def search_project_documents(
        query: str, tool_context: ToolContext
    ) -> dict[str, Any]:
        """Search the project's RAG corpus and return ranked chunks.

        Args:
            query: A self-contained natural-language search query in the
                project's working language. Pronouns and references must
                already be resolved by the caller.

        Returns:
            On hit: {"status": "ok", "chunks": [{"idx", "filename",
            "page_start", "page_end", "text"}, ...]}.
            On miss: {"status": "no_results", "chunks": []}.
            The LLM places `[idx]` markers inline in its answer; the
            aggregator in chats.py renumbers them globally after the run.
        """
        response = await rag.async_retrieve_contexts(
            text=query,
            rag_resources=[rag.RagResource(rag_corpus=corpus_name)],
            rag_retrieval_config=rag.RagRetrievalConfig(top_k=top_k),
        )
        contexts = response.contexts.contexts
        if not contexts:
            return {"status": "no_results", "chunks": []}

        # Per-turn citation index. The aggregator in chats.py renumbers
        # globally after the run (handles dedup across multiple
        # rag_specialist calls in one orchestrator turn).
        citations: list[dict] = tool_context.state.setdefault("citations", [])
        next_idx = len(citations) + 1

        out_chunks: list[dict] = []
        for ctx in contexts:
            text = ctx.text or ""
            pages = [int(m.group(1)) for m in _PAGE_RE.finditer(text)]
            fig = _FIGURE_RE.search(text)

            record = {
                "idx": next_idx,
                "uri": getattr(ctx, "source_uri", None),
                "filename": getattr(ctx, "source_display_name", None)
                or getattr(ctx, "source_uri", None),
                "page_start": pages[0] if pages else None,
                "page_end": pages[-1] if pages else None,
                "figure_label": f"Abb. {fig.group(1)}" if fig else None,
                "score": getattr(ctx, "score", None),
                "snippet": text.strip()[:200],
            }
            citations.append(record)
            out_chunks.append(
                {
                    "idx": next_idx,
                    "filename": record["filename"],
                    "page_start": record["page_start"],
                    "page_end": record["page_end"],
                    "text": text,
                }
            )
            next_idx += 1

        # tool_context.state writes are committed via state delta when
        # the runner processes this tool's response. `setdefault` returned
        # a live ref, so we don't need to re-assign.
        return {"status": "ok", "chunks": out_chunks}

    return FunctionTool(func=search_project_documents)
```

**Notes for the implementer:**
- Do NOT import `VertexAiRagRetrieval` — the subclass approach was proven dead in T0 probe 2.
- The `query: str, tool_context: ToolContext` signature: ADK auto-detects `tool_context` by name and injects it; only `query` ends up in the function declaration sent to the LLM. (Verified in T0 probe 4 dispatch test — same pattern works for a tool with multiple non-context args.)
- Top-k defaults to 10; mirror existing 18.3 behaviour. Tunable per call site by passing `top_k=` to the factory.
- `figure_label` survives in `state["citations"]` for the chip renderer but is not echoed back in `out_chunks` (the LLM doesn't need it for `[N]` placement; rendering reads from session state via the aggregator).

```python
# backend/app/adk/app_factory.py — new
"""Per-corpus AdkApp factory.

One AdkApp per project corpus, cached. Each AdkApp owns its own
InMemorySessionService — sessions do NOT cross instances under in-memory
storage (T0 probe 6). That is fine because strategy (c) replays Supabase
rows into a fresh per-turn session anyway. A migration to
VertexAiSessionService is a separate phase; when it lands, sessions will
share state across rebuilds via the external service.

`app_name` is hard-coded by `AdkApp` to "default-app-name" (T0 probe 7) —
not configurable via constructor.
"""
from __future__ import annotations

import asyncio
from collections import OrderedDict

from vertexai.preview.reasoning_engines import AdkApp

from .agents import make_chat_orchestrator


_MAX_APPS = 256

_apps: "OrderedDict[str, AdkApp]" = OrderedDict()
_locks: dict[str, asyncio.Lock] = {}
_global_lock = asyncio.Lock()


async def get_or_build_app(corpus_name: str) -> AdkApp:
    # Hot path: cached.
    if corpus_name in _apps:
        _apps.move_to_end(corpus_name)
        return _apps[corpus_name]

    # Cold path: serialize per-corpus to avoid double-build.
    async with _global_lock:
        lock = _locks.setdefault(corpus_name, asyncio.Lock())

    async with lock:
        if corpus_name in _apps:
            _apps.move_to_end(corpus_name)
            return _apps[corpus_name]

        app = AdkApp(agent=make_chat_orchestrator(corpus_name))
        # Force runner / session_service / artifact_service initialisation
        # so callers can rely on `_tmpl_attrs["session_service"]` being
        # present on the very first turn.
        app.set_up()
        _apps[corpus_name] = app
        _apps.move_to_end(corpus_name)

        if len(_apps) > _MAX_APPS:
            # Drop oldest. In-flight requests still hold a reference,
            # so eviction is GC-safe — research §4.1.
            _apps.popitem(last=False)

        return app
```

**8c. History replay helper** — converts Supabase `chat_messages` rows into ADK `Event`s seeded into a fresh per-turn session.

```python
# backend/app/adk/history.py — new
"""Replay Supabase chat_messages into a fresh InMemory ADK session.

Per turn, we build a session, append historical events, then run. This
keeps Supabase as the source of truth for chat history (research §2.5
strategy (c)) at the cost of multi-turn tool-call/response fidelity —
prior tool calls become opaque text. We're already paying that cost
under Pattern A.

Verified pattern (T0 probe 5):
  1. await app.async_create_session(user_id=) -> dict with "id"
  2. live = await sess_service.get_session(app_name=, user_id=, session_id=)
  3. await sess_service.append_event(live, Event(author=, content=...))
  4. await app.async_stream_query(message=, user_id=, session_id=)
"""
from __future__ import annotations

import asyncio

from google.adk.events import Event
from google.adk.sessions import Session
from google.genai import types
from vertexai.preview.reasoning_engines import AdkApp

from app.db import supabase


_HISTORY_LIMIT = 20


def _row_to_event(row: dict) -> Event | None:
    role = row["role"]
    if role not in ("user", "assistant"):
        return None
    return Event(
        author="user" if role == "user" else "chat_orchestrator",
        content=types.Content(
            role="user" if role == "user" else "model",
            parts=[types.Part.from_text(text=row["content"] or "")],
        ),
    )


def load_history_rows(
    chat_id: str, user_id: str, limit: int = _HISTORY_LIMIT
) -> list[dict]:
    res = (
        supabase()
        .table("chat_messages")
        .select("id,role,content,created_at")
        .eq("chat_id", chat_id)
        .eq("user_id", user_id)
        .order("created_at", desc=True)
        .limit(limit)
        .execute()
    )
    rows = res.data or []
    rows.reverse()  # oldest first
    return rows


async def seed_session(*, app: AdkApp, user_id: str, chat_id: str) -> Session:
    """Create a new in-memory session, append historical events, return it."""
    # T0 probe 7: AdkApp does not expose session_service / app_name as
    # public attrs. They live in `_tmpl_attrs`. The Runner exposes them
    # publicly but we go through `_tmpl_attrs` to keep one canonical
    # access pattern across the codebase. Pin google-adk version range
    # tightly so this private path doesn't shift under us.
    sess_service = app._tmpl_attrs["session_service"]
    app_name = app._tmpl_attrs["app_name"]

    sess_dict = await app.async_create_session(user_id=user_id)
    session_id = sess_dict["id"] if isinstance(sess_dict, dict) else sess_dict.id

    live: Session = await sess_service.get_session(
        app_name=app_name, user_id=user_id, session_id=session_id
    )

    rows = await asyncio.to_thread(load_history_rows, chat_id, user_id)
    for r in rows:
        evt = _row_to_event(r)
        if evt is not None:
            await sess_service.append_event(live, evt)
    return live
```

```python
        session_id=session.id,

    final_citations, remap = dedupe_and_renumber(raw_citations)
    raw_answer = "".join(answer_parts)
    annotated = rewrite_refs(raw_answer, remap)

        yield f"data: {json.dumps({'type': 'done', 'message_id': msg['id']})}\n\n"
    else:
```

**T9b: event translator** at `backend/app/adk/event_translator.py` — bodies derived from T0 probe 1 (events are dicts; no `kind` field; discriminate from `content.role` + part shape).

```python
# backend/app/adk/event_translator.py — new
"""Translate AdkApp.async_stream_query event dicts into our internal
discriminators. Events are JSON-serialised dicts (T0 probe 1).

Event shapes seen in probe 1:
  - Model text response:
      {"author": <agent>, "content": {"role": "model",
                                       "parts": [{"text": "..."}]},
       "finish_reason": "STOP", ...}
  - Tool call (model -> tool):
      {"author": <agent>, "content": {"role": "model",
                                       "parts": [{"function_call":
                                         {"id": "...", "name": "<tool>",
                                          "args": {...}}}]},
       "finish_reason": "STOP", ...}
  - Tool response (tool -> model):
      {"author": <agent>, "content": {"role": "user",
                                       "parts": [{"function_response":
                                         {"id": "...", "name": "<tool>",
                                          "response": {...}}}]}, ...}

`author` is at the top level. `actions.state_delta` holds tool_context
state writes (e.g. our citations). No explicit "final" event signal —
the stream ends when the runner finishes; downstream code should
accumulate text deltas as it goes.
"""
from __future__ import annotations

from typing import Any


def _parts(event: dict) -> list[dict]:
    return ((event.get("content") or {}).get("parts") or [])


def event_role(event: dict) -> str | None:
    return (event.get("content") or {}).get("role")


def event_kind(event: dict) -> str:
    """Discriminator for SSE forwarding. Returns one of:
    "model_text" | "tool_call" | "tool_response" | "other".
    """
    role = event_role(event)
    parts = _parts(event)
    if role == "model":
        if any("function_call" in p for p in parts):
            return "tool_call"
        if any("text" in p and p.get("text") for p in parts):
            return "model_text"
    elif role == "user":
        if any("function_response" in p for p in parts):
            return "tool_response"
    return "other"


def event_author(event: dict) -> str | None:
    return event.get("author")


def event_text(event: dict) -> str:
    return "".join(p.get("text") or "" for p in _parts(event))


def is_v2_handoff(event: dict) -> bool:
    """True iff this is a tool_response from `run_projektanalyse_v2` whose
    response body carries the {"hand_off": "projektanalyse_v2"} sentinel."""
    if event_kind(event) != "tool_response":
        return False
    for p in _parts(event):
        fr = p.get("function_response") or {}
        if fr.get("name") == "run_projektanalyse_v2" and (
            (fr.get("response") or {}).get("hand_off") == "projektanalyse_v2"
        ):
            return True
    return False


def event_state_delta(event: dict) -> dict[str, Any]:
    return ((event.get("actions") or {}).get("state_delta") or {})
```

```python
import re
_REF_RE = re.compile(r"\[(\d+)\]")


def dedupe_and_renumber(raw: list[dict]) -> tuple[list[dict], dict[int, int]]:
    seen: dict[tuple, int] = {}
    final: list[dict] = []
    remap: dict[int, int] = {}
    for c in raw:
        key = (c["uri"], c.get("page_start"), (c.get("snippet") or "")[:80])
        if key in seen:
            remap[c["idx"]] = seen[key]
            continue
        new_idx = len(final) + 1
        seen[key] = new_idx
        remap[c["idx"]] = new_idx
        final.append({**c, "idx": new_idx})
    return final, remap


def rewrite_refs(text: str, remap: dict[int, int]) -> str:
    return _REF_RE.sub(lambda m: f"[{remap.get(int(m.group(1)), m.group(1))}]", text)
```

## Migration risk note

19.0 trades a small, well-understood architecture (single Vertex RAG call, byte-offset citation splicing) for a more capable but less-tested one (multi-agent tree, query rephrasing, structured citations, per-corpus factory). Dominant risk classes:

1. **Latency regression**. Pro orchestrator + Flash sub-agents adds ≥ 1 round-trip per turn. Benchmark in T15b. Worst case for self-contained questions: ~1.5–2× current first-delta latency.
2. **Orchestrator misroutes** (e.g. dispatches to web_researcher when answer is in project docs). T14 eval covers explicit routing decisions; failures point to instruction-tightening.
3. **Rephrasing loses precision**. Orchestrator may rewrite "und das?" to a query that doesn't match chunk vocabulary. Mitigation: rephrasing rules require Eigenname/concept tokens to survive. T14 `pronoun_resolution_*` cases lock this down.
4. ~~**`after_tool_callback` doesn't fire for VertexAiRagRetrieval (issue #2629).**~~ Moot — we no longer subclass `VertexAiRagRetrieval`. Plain FunctionTool callbacks always fire.
5. ~~**Multi-question dispatch is serialized, not parallel**.~~ T0 probe 4 confirmed PARALLEL (start gap 0.000s, overlap 1.005s). T7 instruction can rely on parallel fan-out; T14's `multi_question_split` keeps the parallel timing assertion.
6. **InMemorySessionService session creation overhead at scale**. Per-turn session create + N event appends. Scales with history length. If hot-path latency dominates, switch to (a) Vertex sessions earlier than planned.
7. **AdkApp construction time on cold cache**. Research §4.3: no published benchmark; expect ~ms for object init + first network call to Gemini on warmup. Acceptable.
8. **Token billing surprise**. N parallel rag_specialist runs = N Flash turns + 1 Pro turn. Watch the post-T15 cost report.
9. **Private-attribute access (`app._tmpl_attrs["session_service"]`, `app._tmpl_attrs["app_name"]`) breaks on ADK upgrade.** Pin the `google-adk` version range tightly (`>=1.16,<2.0`, currently 1.31.1). On upgrade, re-run T0 probes 5 + 7 to confirm the access path still works; if Google promotes `app.session_service` and `app.app_name` to public, switch.

Diagnosis recipe post-19.0:

1. Bad routing → tighten orchestrator's ROUTING-ENTSCHEIDUNG; add eval case
2. Bad rephrasing → tighten UMFORMULIERUNGS-REGELN; add eval case
3. Citations off-by-one or duplicated → bug in `dedupe_and_renumber`; fix in `citation_aggregator.py`
4. v2 fires unexpectedly → tighten NO-V2-ESCALATION in orchestrator; if persistent, code-level guard in T9 SSE translator
5. Aggregation/sum hallucinations → tighten NO-SELF-SUM in rag_specialist; never add code-side compute logic
6. Pages missing on chips → check chunk had `[Seite N]` in LangSmith trace; if 11% is now 25%, parser regressed and fix is upstream
7. Latency regression → first check if a rag_specialist call is happening when the answer was in history (orchestrator misclassified)

Re-run the 18.0.1 benchmark battery after 19.0 lands. Quality ≥ 18.3 baseline is the bar; latency may grow up to ~2× for follow-up turns, acceptable in exchange for correct rephrasing.
