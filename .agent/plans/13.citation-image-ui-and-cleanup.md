# 13. Citation + Image UI, OpenAI Cleanup

## Complexity: ⚠️ Medium

Frontend-heavy plan: render citation chips inline, render technical-drawing images attached to assistant messages, and provide a "open source PDF at page N" viewer. Then we rip out the OpenAI integration completely.

This is the user-visible payoff of plans 10–12.

---

## Goal

After this plan:

- Every assistant message displays clickable citation chips (`[pumpe-handbuch.pdf p.17]`).
- When the assistant references a figure, the figure image renders inline with the answer.
- Clicking a chip opens a PDF viewer dialog at the cited page.
- All OpenAI code, columns, env vars, and the `openai` Python package are removed. The codebase has a single LLM provider: Gemini.

### 13.1 — Citation chip component

**File:** `/home/lukasthomas/sleek-rag/components/citation-chip.tsx` (new)

Compact, clickable badge. Variant for figures (with figure label) vs plain page citations.

```tsx
import { FileText, ImageIcon } from "lucide-react";
import { Badge } from "@/components/ui/badge";

type Citation = {
  chunk_id: string;
  file_id: string;
  filename: string;
  page_start: number;
  page_end: number;
  figure_label: string | null;
  image_path: string | null;
};

export function CitationChip({
  citation, index, onClick,
}: { citation: Citation; index: number; onClick: () => void }) {
  const Icon = citation.figure_label ? ImageIcon : FileText;
  const pageLabel = citation.page_start === citation.page_end
    ? `p.${citation.page_start}`
    : `p.${citation.page_start}-${citation.page_end}`;
  return (
    <button
      onClick={onClick}
      className="inline-flex items-center gap-1 text-xs"
    >
      <Badge variant="secondary" className="hover:bg-accent cursor-pointer">
        <Icon className="size-3 mr-1" />
        [{index}] {citation.filename} {pageLabel}
        {citation.figure_label && ` · ${citation.figure_label}`}
      </Badge>
    </button>
  );
}
```

```tsx
function FigureThumb({ citation, onOpen }: { citation: Citation; onOpen: () => void }) {
  const [url, setUrl] = useState<string | null>(null);
  useEffect(() => {
    if (!citation.image_path) return;
    const supabase = createClient();
    supabase.storage.from("chunk-images")
      .createSignedUrl(citation.image_path, 600)
      .then(({ data }) => setUrl(data?.signedUrl ?? null));
  }, [citation.image_path]);
  if (!url) return null;
  return (
    <button onClick={onOpen} className="block mt-3 rounded-lg overflow-hidden border hover:ring-2 hover:ring-primary">
      <img src={url} alt={citation.figure_label ?? "figure"}
           className="max-w-md max-h-80 object-contain bg-muted" />
      <div className="text-xs text-muted-foreground p-1.5 text-left">
        {citation.figure_label ?? "Figure"} — {citation.filename} p.{citation.page_start}
      </div>
    </button>
```

### 13.3 — PDF viewer dialog

**File:** `/home/lukasthomas/sleek-rag/components/pdf-viewer-dialog.tsx` (new)

Open the source PDF at a specific page. Use `react-pdf` (already a common Next.js dep) or `iframe` with browser-native PDF viewer + `#page=N` fragment.

Simpler v1 — iframe with `#page=N`:

```tsx
import { Dialog, DialogContent, DialogHeader, DialogTitle } from "@/components/ui/dialog";
import { useEffect, useState } from "react";
import { createClient } from "@/lib/supabase/client";

export function PdfViewerDialog({
  citation, open, onOpenChange,
}: { citation: Citation | null; open: boolean; onOpenChange: (v: boolean) => void }) {
  const [url, setUrl] = useState<string | null>(null);

  useEffect(() => {
    if (!citation) return;
    // Resolve gcs_blob_path of the source file via project-files bucket
    const supabase = createClient();
    supabase.from("project_files").select("gcs_blob_path").eq("id", citation.file_id).single()
      .then(({ data }) => {
        if (!data?.gcs_blob_path) return;
        supabase.storage.from("project-files")
          .createSignedUrl(data.gcs_blob_path, 600)
          .then(({ data: sig }) => setUrl(`${sig?.signedUrl}#page=${citation.page_start}`));
      });
  }, [citation]);

  return (
    <Dialog open={open} onOpenChange={onOpenChange}>
      <DialogContent className="max-w-5xl h-[85vh]">
        <DialogHeader>
          <DialogTitle>
            {citation?.filename} — p.{citation?.page_start}
            {citation?.figure_label && ` · ${citation.figure_label}`}
          </DialogTitle>
        </DialogHeader>
        {url ? (
          <iframe src={url} className="w-full h-full rounded border" />
        ) : (
          <div className="flex-1 grid place-items-center text-muted-foreground">Loading…</div>
        )}
      </DialogContent>
    </Dialog>
  );
}
```

Wire from `chat.tsx`:

**Validation:** click a chip → dialog opens, PDF renders, scrolled to the right page (works in Chrome/Firefox; Safari ignores `#page=` and opens at page 1, document this as a known limitation).

### 13.4 — Inline citation markers in message text

The LLM already emits `[1]` / `[2]` markers in its prose (per the system prompt in plan 12). Make these clickable.

In `chat.tsx` Markdown rendering, post-process via a remark plugin or simple string replace before passing to `react-markdown`. Simplest version:

```tsx
function linkifyCitations(text: string, citations: Citation[]) {
  return text.replace(/\[(\d+)\]/g, (m, n) => {
    const idx = parseInt(n, 10) - 1;
    if (idx < 0 || idx >= citations.length) return m;
    return `[${m}](#cite-${citations[idx].chunk_id})`;
  });
}

// In the renderer:
const renderedContent = message.citations
  ? linkifyCitations(message.content, message.citations)
  : message.content;
// Custom <a> renderer in react-markdown handles the click → setViewerCitation
```
