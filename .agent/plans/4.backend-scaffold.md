# 4. Backend Scaffold

✅ **Simple** — Single-pass executable, low risk.

## Context

The kickoff plan (`3.module-1-kickoff.md`) commits to a Python + FastAPI backend in `backend/` per CLAUDE.md. Nothing exists yet — no `backend/` folder, no `venv/`, no Python files. This sub-plan delivers the empty-but-runnable shell that subsequent sub-plans hang routes off of. No external credentials are needed at this stage; the root `.env.example` documents every key (frontend + backend) so the user populates a single `.env` at project root.

## Goal

Running `uvicorn app.main:app --reload --port 8000` from `backend/` (with the venv activated) returns 200 on `GET /health` and accepts CORS preflights from `http://localhost:3000`. That's it.

### 4.3 `backend/app/config.py`

Pydantic-Settings model that loads from the **repo-root** `.env` (resolved by absolute path so it works from any cwd). Define stub fields for all M1 keys so later sub-plans only add usage, not config plumbing:

```python
from pathlib import Path
from pydantic_settings import BaseSettings, SettingsConfigDict

ROOT_ENV = Path(__file__).resolve().parents[2] / ".env"

class Settings(BaseSettings):
    model_config = SettingsConfigDict(env_file=str(ROOT_ENV), extra="ignore")

    cors_origins: str = "http://localhost:3000"

    # Filled in by later sub-plans
    openai_api_key: str = ""
    supabase_url: str = ""
    supabase_service_role_key: str = ""
    supabase_jwt_secret: str = ""
    langsmith_api_key: str = ""
    langsmith_project: str = "sleek-rag"

settings = Settings()
```

**Validation:** `python -c "from app.config import settings, ROOT_ENV; print(ROOT_ENV); print(settings.cors_origins)"` prints the absolute path to the repo-root `.env` and the cors default — no error even when `.env` is missing.

### 4.4 `backend/app/main.py`

```python
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware

from app.config import settings

app = FastAPI(title="sleek-rag backend")

app.add_middleware(
    CORSMiddleware,
    allow_origins=[origin.strip() for origin in settings.cors_origins.split(",")],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

@app.get("/health")
def health():
    return {"status": "ok"}
```

**Validation:** `curl http://localhost:8000/health` → `{"status":"ok"}`; `curl -i -X OPTIONS http://localhost:8000/health -H "Origin: http://localhost:3000" -H "Access-Control-Request-Method: GET"` → `200` with `access-control-allow-origin: http://localhost:3000`.

### 4.5 Root `.env.example`

# --- Backend (server-only; never exposed to browser) ---

SUPABASE_JWT_SECRET=

Pydantic-Settings uses defaults for missing keys, so no real `.env` file is required to start the dev server.
