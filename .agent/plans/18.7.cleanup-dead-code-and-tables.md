### Task 2 — Drop Supabase tables and RPCs

**What:**

New migration `0021_drop_legacy_retrieval_schema.sql`:

```sql
-- Drop the embedding-pipeline tables (superseded by Vertex AI RAG Engine corpora)
drop table if exists public.chunk_images cascade;
drop table if exists public.document_chunks cascade;
drop table if exists public.ingest_jobs cascade;

-- Drop the now-orphaned RPCs
drop function if exists public.match_chunks(uuid, vector, int, float);
drop function if exists public.match_chunks_hybrid(uuid, vector, text, int, float);
drop function if exists public.match_chunks_hybrid_with_heading(uuid, vector, text, int, float);
drop function if exists public.search_chunks_filtered(uuid, vector, text, int, float, text[]);
drop function if exists public.chunks_in_range(uuid, uuid, int, int, text);
drop function if exists public.chunks_on_page(uuid, uuid, int, int);
drop function if exists public.document_outline(uuid, uuid);
drop function if exists public.claim_next_ingest_job();

-- (HNSW + GIN indexes drop automatically with their tables)
```

**Files:**
- `supabase/migrations/0021_drop_legacy_retrieval_schema.sql` — new
- Apply via Supabase MCP (`apply_migration`) or `supabase db push`

**Notes:**
- Confirm no live RLS policy or other migration depends on these tables before dropping. Run `supabase db diff` if uncertain.
- Old historical migrations stay in the `supabase/migrations/` folder — we don't rewrite history. New environments will apply 0001..0021 in order; the legacy schema is created and then dropped within the migration chain.

**Validation:**
- Migration applies cleanly forward (against a fresh DB and against the current dev DB)
- After apply, querying `select * from document_chunks limit 1` errors with relation-doesn't-exist
- All chat / projektanalyse / file-upload flows still work
- `pytest backend/tests/` → all green

### Task 3 — Remove unused Python deps

**What:**

After Task 1, scan `pyproject.toml` for deps that no longer have any importer:

```bash
for dep in $(grep -oP '"\K[a-z0-9_-]+(?=>)' pyproject.toml); do
  count=$(grep -rln "^import $dep\|^from $dep" backend/ --include="*.py" | wc -l)
  echo "$count $dep"
done | sort -n | head -20
```

Likely candidates for removal:
- `pgvector` Python adapter (if any) — vector ops gone
- Any rerank / cross-encoder client (if separate from `ranking_client`)
- Legacy embedding clients

Keep:
- `google-cloud-aiplatform` (RAG Engine + Vertex AI), `google-genai` (Gemini SDK), `pymupdf` (view_document_page), `google-cloud-storage` (GCS)
- `supabase`, `pydantic`, `fastapi`, `uvicorn` — core stack
- LibreOffice subprocess helper (still needed for DOCX/XLSX → PDF)

After trimming:
```bash
uv lock --upgrade  # or pip-compile
uv sync            # install lock state
```

**Validation:**
- `pip check` reports no dependency conflicts
- `pytest backend/tests/` passes
- Backend boots cleanly
