# 18.3 — Chat loop replacement (Pattern A, Vertex RAG grounding tool)

**Complexity:** 🔴 Complex (load-bearing — biggest behavior change in the migration)
**Prerequisites:** [18.2 Ingest pipeline](18.2.ingest-pipeline-replacement.md)
**Companion:** [18.0 Master Overview](18.0.vertex-rag-migration-master.md)

---

## Goal

Replace the entire custom chat loop (~600 LOC of orchestration in `chats.py` + `sufficiency.py` + `answer_verifier.py` + section/outline tools) with **Pattern A: Vertex RAG grounding tool inside a Gemini chat session**. The result mirrors the user's vanilla JS test verbatim — same model, same config, same tool surface — and lets Gemini handle retrieval, ranking, and answer composition end-to-end.

Domain rules (SIA scoping, no self-sum, role policy, no-v2-auto-escalation) move from code-enforced gates into the **`system_instruction`** field. The chat loop becomes a thin streaming dispatcher (~80 LOC) that:

1. Builds a chat session with grounding tool + 3 function tools (view_document_page, run_projektanalyse, run_projektanalyse_v2)
2. Streams deltas
3. Handles function calls (view_document_page inline; projektanalyse handlers break out to dedicated streamers)
4. Parses `grounding_metadata` for citations after the stream completes

After 18.3, large parts of the test suite go away (sufficiency, verifier, force-tool, outlined_file_ids, multi-tool dispatch tests). What replaces them: a small set of tests on grounding-citation extraction + system-prompt content checks.

---

## What goes away (and why it's safe)

| Component | Why it's safe to drop |
|---|---|
| `sufficiency.py` | Pattern A doesn't expose chunks to the backend before the answer; sufficiency rating is structurally impossible. Domain rules ("if evidence is insufficient, say so") move into `system_instruction` |
| `answer_verifier.py` | Same reason — no chunks to verify against. Rule moves into prompt: "Don't fabricate. If unsure, say so." |
| `force_tool_next_iter` + `force_tool_next_iter_target` | No second-round retrieval; Gemini auto-retrieves as much as it needs |
| `outlined_file_ids` gate | `read_section`/`list_document_outline` go away entirely (next row) |
| `tools/section.py` (read_section) | Pattern A retrieval surfaces section content automatically when the query references a section |
| `tools/outline.py` (list_document_outline) | Same — outlines come from the model's reading of grounded text |
| `ranking_client.py` callers | RAG Engine ranks internally |
| 3-error-streak bail | No retries to fail; one stream per turn |
| Multi-tool parallel dispatch + `ref_offset` accounting | Citations come from `grounding_metadata`, not assembled from multiple tool outputs |
| `_build_system_message` file-inventory block | Pattern A doesn't need a file inventory in the prompt — the corpus is the inventory. Optional: keep a one-liner ("This project has N files") if Gemini benefits |

**No-v2-auto-escalation enforcement moves to prompt-only.** The hard code-level guarantee from `chats.py:371-375` and the regression test at `test_chat_force_tool.py:260-267` go away. Replacement: an explicit instruction in `system_instruction`. If the model goes rogue, it's a prompt fix, not a code fix.

### Task 4 — `backend/app/vertex_rag_grounding.py` helper

**What:**

A tiny module that builds the grounding tool config for a given project's corpus. Mirrors the user's JS test verbatim.

```python
"""Build the Vertex RAG retrieval grounding tool for a project's corpus."""

from typing import Optional
from google.genai import types
from backend.app.db import supabase


async def grounding_tool_for_project(project_id: str) -> Optional[types.Tool]:
    """Returns the grounding Tool, or None if the project has no corpus yet."""
    row = await supabase.table("projects") \
        .select("rag_corpus_name").eq("id", project_id).single().execute()
    corpus = (row.data or {}).get("rag_corpus_name")
    if not corpus:
        return None
    return types.Tool(
        retrieval=types.Retrieval(
            vertex_rag_store=types.VertexRagStore(
                rag_resources=[types.VertexRagStoreRagResource(rag_corpus=corpus)]
            )
            # No rag_retrieval_config — defaults match the user's test
        )
    )
```

### Task 6 — Replace `chats.py` chat loop with Pattern A streamer

**What:**

`backend/app/routers/chats.py::_send_message_stream` (~600 LOC today) shrinks to roughly this:

```python
import json
from google import genai
from google.genai import types
from backend.app.config import settings
from backend.app.db import supabase
from backend.app.system_instructions import SYSTEM_INSTRUCTION
from backend.app.vertex_rag_grounding import grounding_tool_for_project
from backend.app.tools.view_page import view_document_page_executor, VIEW_DOCUMENT_PAGE_DECL
from backend.app.projektanalyse import (
    PROJEKTANALYSE_DECL, PROJEKTANALYSE_V2_DECL,
    stream_projektanalyse, stream_projektanalyse_v2,
)


_client = genai.Client(vertexai=True, project=settings.gcp_project_id, location=settings.gcp_location)


def _gen_config(project_id: str) -> types.GenerateContentConfig:
    """Mirrors the user's vanilla JS test config exactly."""
    grounding = grounding_tool_for_project(project_id)  # None if no files yet
    function_tools = types.Tool(function_declarations=[
        VIEW_DOCUMENT_PAGE_DECL,
        PROJEKTANALYSE_DECL,
        PROJEKTANALYSE_V2_DECL,
    ])
    tools = [t for t in [grounding, function_tools] if t is not None]
    return types.GenerateContentConfig(
        max_output_tokens=65535,
        temperature=1.0,
        top_p=0.95,
        thinking_config=types.ThinkingConfig(thinking_level="HIGH"),
        safety_settings=[
            types.SafetySetting(category=c, threshold="OFF") for c in [
                "HARM_CATEGORY_HATE_SPEECH",
                "HARM_CATEGORY_DANGEROUS_CONTENT",
                "HARM_CATEGORY_SEXUALLY_EXPLICIT",
                "HARM_CATEGORY_HARASSMENT",
            ]
        ],
        system_instruction=SYSTEM_INSTRUCTION,
        tools=tools,
    )


async def _load_history(chat_id: str) -> list[types.Content]:
    rows = await supabase.table("chat_messages") \
        .select("role, content") \
        .eq("chat_id", chat_id) \
        .order("created_at") \
        .limit(20) \
        .execute()
    return [
        types.Content(role=r["role"], parts=[types.Part.from_text(r["content"])])
        for r in rows.data
    ]


async def _send_message_stream(*, chat: dict, text: str, chat_id: str, user_id: str, template):
    project_id = chat["project_id"]

    # Persist user message first (existing logic)
    await _persist_user_message(chat_id, user_id, text)

    history = await _load_history(chat_id)
    config = await _gen_config_async(project_id)

    chat_session = _client.aio.chats.create(
        model=settings.vertex_rag_chat_model,  # "gemini-2.5-flash"
        config=config,
        history=history[:-1],  # exclude the user msg we just appended; pass it as new_message
    )

    answer_parts = []
    grounding_chunks_seen = []
    final_response = None

    async for chunk in chat_session.send_message_stream(text):
        # Stream text deltas as SSE
        if chunk.text:
            answer_parts.append(chunk.text)
            yield _sse_delta(chunk.text)

        # Detect function calls
        function_calls = _extract_function_calls(chunk)
        for fc in function_calls:
            if fc.name == "view_document_page":
                result = await view_document_page_executor(
                    args=dict(fc.args), project_id=project_id, user_id=user_id
                )
                # Send tool response back; chat session will continue
                # (Gemini auto-resumes after function_response)
                # Multi-modal Part for the image is handled inline if present
                continue  # The SDK threads function responses via chat session state
            elif fc.name == "run_projektanalyse":
                # Hand off to dedicated streamer; abort this chat-session stream
                async for ev in stream_projektanalyse(
                    template=template, chat_id=chat_id, user_id=user_id
                ):
                    yield ev
                return
            elif fc.name == "run_projektanalyse_v2":
                async for ev in stream_projektanalyse_v2(
                    template=template, chat_id=chat_id, user_id=user_id
                ):
                    yield ev
                return

        final_response = chunk  # last chunk holds final grounding_metadata

    # After stream completes — extract citations from grounding_metadata
    citations = _grounding_to_citations(final_response, project_id)
    yield _sse_meta(citations=citations)

    # Persist assistant message
    full_text = "".join(answer_parts)
    msg_id = await _persist_assistant_message(chat_id, user_id, full_text, citations)
    yield _sse_done(message_id=msg_id)
```

### Task 7 — Citation extraction from `grounding_metadata`

**What:**

`backend/app/citations.py`:

```python
"""Map grounding_metadata.grounding_chunks → frontend Citation shape with regex enrichment."""

import hashlib
import re
from typing import Optional
from backend.app.db import supabase


_PAGE_RE = re.compile(r"\[Seite (\d+)\]")
_FIGURE_RE = re.compile(r"\[Abb\.?\s*(\d+(?:\.\d+)*):\s*([^\]]+)\]")


async def grounding_to_citations(response, project_id: str) -> list[dict]:
    if not response or not getattr(response, "candidates", None):
        return []
    meta = getattr(response.candidates[0], "grounding_metadata", None)
    if not meta or not getattr(meta, "grounding_chunks", None):
        return []

    files_by_uri = await _files_by_gcs_uri(project_id)
    citations = []
    for chunk in meta.grounding_chunks:
        ctx = getattr(chunk, "retrieved_context", None)
        if not ctx:
            continue
        text = getattr(ctx, "text", "") or ""
        page_starts = [int(m.group(1)) for m in _PAGE_RE.finditer(text)]
        figure_match = _FIGURE_RE.search(text)
        file_row = files_by_uri.get(ctx.uri, {})
        citations.append({
            "chunk_id": hashlib.sha1(f"{ctx.uri}|{text[:200]}".encode()).hexdigest()[:16],
            "file_id": file_row.get("id"),
            "filename": file_row.get("filename") or getattr(ctx, "title", None) or ctx.uri,
            "page_start": page_starts[0] if page_starts else None,
            "page_end": page_starts[-1] if page_starts else None,
            "snippet": text[:200],
            "figure_label": f"Abb. {figure_match.group(1)}" if figure_match else None,
            "image_path": None,  # dropped per Q2
            "score": None,        # not exposed by grounding API in the same way
        })
    return citations


async def _files_by_gcs_uri(project_id: str) -> dict[str, dict]:
    rows = await supabase.table("project_files") \
        .select("id, filename, gcs_blob_path") \
        .eq("project_id", project_id).execute()
    return {r["gcs_blob_path"]: r for r in (rows.data or [])}
```

### Task 8 — Frontend SSE handling: meta frame at end

**What:**

Today citations arrive in a `meta` SSE frame **before** delta frames (per the frontend map: "type 'meta' for citations, type 'delta' for content"). With Pattern A, citations come from `grounding_metadata` which is only fully populated **after** the stream completes.

Two options:
- **A1:** Buffer the entire stream, emit meta first, then deltas. Loses streaming UX.
- **A2:** Stream deltas as they come, emit meta at the end. **Recommended.** One-line frontend tweak: the citation-rendering block becomes "render once meta frame arrives, regardless of order".

Update `components/chat.tsx` and `components/app-shell.tsx:935-942`:
- Citation parsing already keys off `payload.type === "meta"` and accumulates citations
- The existing dedup logic (`chunk_id` based) doesn't care about ordering
- Confirm citation chips render after the answer text, not blocking the stream

## Migration risk note

This is the load-bearing plan. **Quality-related regressions found here are most likely prompt issues, not code issues.** The recipe for diagnosing problems post-18.3:

1. **Bad answer on aggregation question** ("wie viele Etappen?") → tighten the aggregation clause in `system_instruction`
2. **Inversion / fabrication** → tighten the honesty clause; consider escalating chat model from Flash → Pro
3. **v2 called when not asked** → tighten the no-v2 clause; if persistent, add a code-level pre-call guard as a fallback
4. **Citations missing page numbers** → check whether LLM Parser's prompt is producing `[Seite N]` markers consistently; tighten parsing prompt
5. **Citations missing entirely** → check `grounding_metadata` is being populated by Vertex (sometimes Gemini answers from training without grounding); add a system_instruction clause forcing it to retrieve

Re-run the 18.0.1 benchmark after 18.3 lands to quantify regressions. The post-18.3 numbers should be ≥ Variant A's pre-migration numbers (since they use the same underlying Pattern A setup).
