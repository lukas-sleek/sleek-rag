# 18.2 — Ingest pipeline replacement

**Complexity:** ⚠️ Medium
**Prerequisites:** [18.1 Bootstrap](18.1.bootstrap-iam-bucket-schema.md)
**Companion:** [18.0 Master Overview](18.0.vertex-rag-migration-master.md)

---

## Goal

Replace the current embedding/parsing/persisting worker (`backend/app/workers/ingest.py`, ~300 LOC) with:

1. A **direct GCS upload** at file-upload time (PDF stored permanently in `gs://sleek-rag-files-{env}/...`)
2. A **`rag.import_files()` LRO trigger** at upload time (RAG Engine handles parsing + chunking + embedding internally with the LLM Parser config)
3. A **thin LRO poller worker** that flips `project_files.status` from `parsing` → `ready` (or `failed`) when the LRO resolves

After this plan, file upload UX is unchanged — the user uploads, sees status transitions in the UI, gets a "ready" status. But under the hood, no chunks live in Postgres, no embeddings are computed by us, no Document AI processor is called. RAG Engine owns the index.

Chat / projektanalyse continue to work against the OLD retrieval path during 18.2 (we don't touch those yet — that's 18.3+). Existing chat tests should still pass; the only thing changing is what powers the corpus.

### Task 1 — `backend/app/rag_corpus.py` helpers

**What:**

A new module that owns all interactions with `vertexai.preview.rag`:

```python
"""Helpers for managing per-project Vertex RAG corpora."""

from typing import Optional
import vertexai
from vertexai.preview import rag
from backend.app.config import settings
from backend.app.parsing_prompts import SIA_PARSING_PROMPT
from backend.app.db import supabase  # existing helper

vertexai.init(project=settings.gcp_project_id, location=settings.gcp_location)


def _embedding_config() -> rag.RagEmbeddingModelConfig:
    return rag.RagEmbeddingModelConfig(
        vertex_prediction_endpoint=rag.VertexPredictionEndpoint(
            publisher_model=f"publishers/google/models/{settings.vertex_rag_embedding_model}"
        )
    )


def _llm_parser_config() -> rag.LlmParserConfig:
    parsing_model = (
        f"projects/{settings.gcp_project_id}"
        f"/locations/{settings.gcp_location}"
        f"/publishers/google/models/{settings.vertex_rag_parsing_model}"
    )
    return rag.LlmParserConfig(
        model_name=parsing_model,
        max_parsing_requests_per_min=settings.vertex_rag_parsing_max_requests_per_min,
        custom_parsing_prompt=SIA_PARSING_PROMPT,
    )


async def ensure_corpus_for_project(project_id: str) -> str:
    """Lazy-create a corpus for this project. Returns rag_corpus_name."""
    row = await supabase.table("projects").select("rag_corpus_name").eq("id", project_id).single().execute()
    if row.data and row.data.get("rag_corpus_name"):
        return row.data["rag_corpus_name"]

    corpus = rag.create_corpus(
        display_name=f"sleek-rag-{project_id}",
        backend_config=rag.RagVectorDbConfig(rag_embedding_model_config=_embedding_config()),
    )
    await supabase.table("projects").update({"rag_corpus_name": corpus.name}).eq("id", project_id).execute()
    return corpus.name


async def import_pdf(corpus_name: str, gcs_uri: str) -> str:
    """Trigger ingestion. Returns the LRO operation name for polling."""
    op = rag.import_files(
        corpus_name,
        [gcs_uri],
        llm_parser=_llm_parser_config(),
        transformation_config=rag.TransformationConfig(
            chunking_config=rag.ChunkingConfig(chunk_size=1024, chunk_overlap=200)
        ),
    )
    return op.operation.name  # adjust to actual SDK shape


async def delete_corpus(corpus_name: str) -> None:
    rag.delete_corpus(corpus_name)
```

### Task 3 — Refactor file upload endpoint

**What:**

`backend/app/routers/files.py` upload endpoint changes:

**Before:**
```
POST /api/projects/{project_id}/files
  → save to Supabase Storage
  → enqueue ingest_jobs row
  → return project_files row (status='pending')
```

**After:**
```
POST /api/projects/{project_id}/files
  → if DOCX/XLSX: LibreOffice convert to PDF (existing helper, untouched)
  → upload PDF bytes to gs://{bucket}/{user_id}/{project_id}/{file_id}/original.pdf
  → ensure_corpus_for_project(project_id) → corpus_name
  → import_pdf(corpus_name, gcs_uri) → returns LRO operation name
  → insert project_files row with:
       status = 'parsing'
       gcs_blob_path = gs://{bucket}/...
       ingest_lro_name = <operation name>   # new column, see Task 4
  → return project_files row
```

### Task 5 — LRO poller worker

**What:**

Replace `backend/app/workers/ingest.py` with a much smaller poller. New file `backend/app/workers/rag_lro_poller.py`:

```python
"""Polls outstanding rag.import_files LROs and updates project_files status."""

import asyncio
import logging
from vertexai.preview import rag
from google.api_core import operations_v1
from backend.app.db import supabase
from backend.app.config import settings

log = logging.getLogger(__name__)

POLL_INTERVAL_SEC = 15


async def _poll_one(row: dict) -> None:
    """Check the LRO and update the row accordingly."""
    op_name = row["ingest_lro_name"]
    file_id = row["id"]

    try:
        # Poll the operation
        # Note: actual SDK shape depends on google-cloud-aiplatform version;
        #       see https://docs.cloud.google.com/vertex-ai/generative-ai/docs/rag-engine/rag-quickstart
        op = rag.get_import_files_operation(op_name)  # OR equivalent

        if not op.done:
            return  # still running, check again next tick

        if op.error:
            await supabase.table("project_files").update({
                "status": "failed",
                "ingest_error": str(op.error),
                "ingest_lro_name": None,
            }).eq("id", file_id).execute()
            log.warning("file %s ingest failed: %s", file_id, op.error)
            return

        # Find the rag_file_name from the result
        result = op.result()
        rag_file_name = result.imported_rag_files_metadata[0].rag_file_id  # adjust to actual SDK field
        await supabase.table("project_files").update({
            "status": "ready",
            "rag_file_name": rag_file_name,
            "ingest_lro_name": None,
        }).eq("id", file_id).execute()
        log.info("file %s ingest ready: %s", file_id, rag_file_name)

    except Exception as e:
        log.exception("poll error for file %s", file_id)


async def run_poller() -> None:
    while True:
        rows = await supabase.table("project_files") \
            .select("id, ingest_lro_name") \
            .eq("status", "parsing") \
            .not_.is_("ingest_lro_name", None) \
            .execute()
        await asyncio.gather(*(_poll_one(r) for r in rows.data))
        await asyncio.sleep(POLL_INTERVAL_SEC)
```

The worker entry point launches `run_poller()` as a background task on FastAPI startup (replace the existing ingest worker startup hook).

**Files:**
- `backend/app/workers/rag_lro_poller.py` — new
- `backend/app/main.py` (or wherever startup hooks live) — replace ingest worker startup with poller

**Validation:**
