# 1. Design Port — claude.ai/design → Next.js

🔴 **Complex** — Many components, drag-drop, modals, mock streaming. Broken into ordered tasks; each task is single-pass executable on its own.

## Goal
Port the entire `sleek-rag.html` prototype (claude.ai/design handoff bundle, located at `/tmp/sleek-design/rag/project/`) into this repo as a working Next.js 16 + TypeScript app. Match the visual output pixel-perfectly. All interactions are mocked client-side. Backend wiring (auth, OpenAI, Supabase, etc.) comes later in module 1+.

## Scope (option 1a from the discussion)
- Login + register screens
- Sidebar with collapsible projects, drag-drop reorder (projects + chats), search, rename/delete via context menu, "Vorlage Analyse" button, "Neues Projekt" button, collapsed compose icon, footer avatar
- Chat view with mock streaming, custom composer (attachments, model selector, sliders/temperature, mic, send), EmptyState (with no-files variant)
- ProjectFilesModal: per-project mock files, 5s analyzing skeleton, drag-drop upload, file-type filter, footer + button
- TemplateAnalysisModal with the 11 prefilled questions, localStorage persistence
- ConfirmDialog, PromptDialog, toast stack, full-screen drag-drop overlay
- All copy German; brand tokens kept (canvas `#212121`, sidebar `#171717`, accent `#ff4539`)

## Out of scope (this plan)
- Real auth, real LLM, real file upload, real persistence beyond localStorage
- Tailwind/shadcn translation of `styles.css` — copy CSS verbatim into `globals.css` per user's instruction ("we will merge the css to tailwind later")
- TweaksPanel — design-tool-only feature that talks to a parent iframe; renders nothing in standalone mode. Port as a no-op stub.

## Stack
- Next.js 16 (App Router) + TypeScript + React 18
- No Tailwind / shadcn installed yet (deferred — to be added when CSS is migrated)
- Single client component `App` mounted at `app/page.tsx`

## Risks / open questions
- The prototype uses React 18 UMD + Babel-standalone. Next.js will use React 18 native — should work identically.
- `'use client'` boundaries: every interactive component touches state, so they all need it. The cleanest is `app-shell.tsx` `'use client'` + all component children imported from it (which inherits client-ness automatically).
- The TweaksPanel postMessage listeners hit `window.parent` — guarded by a no-op stub so it's a non-issue.
- `dangerouslySetInnerHTML` for the inline EAG logo isn't needed; we use the `/public/eichenberger-logo.svg` file via `<img>` like the prototype's final state.
